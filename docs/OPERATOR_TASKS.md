# 운영자 실행 메모

이번 변경 이후 사용자가 직접 해야 할 일:
- 프론트엔드/백엔드 개발 서버 재시작(실행 중이었다면) 후 대시보드 “더 보기” 버튼 동작 확인.
- `.env.local`에서 `NEXT_PUBLIC_USE_MOCK_DATA` 값이 실제 의도대로 설정되어 있는지 확인(실서비스는 false).
- 리더보드 → 유저 지갑 클릭 → 공유 프롬프트 모달에서 카드 클릭 시 상세 페이지로 이동하는지 수동 점검.
- 프롬프트 입력 → 결제 요구 → 승인 후 자동 재시도 → 대화 생성 → 카드 클릭 → 상세 진입까지 오류 없이 동작하는지 확인(특히 402 처리 및 상세 API 호출).
- 공유 직후 새 카드가 안 보이면 필터/정렬을 “전체/최신순”으로 초기화한 뒤 다시 확인(코드가 자동으로 초기화하도록 변경됨).
- 업적 시드/자동 획득 반영을 위해 `cd backend && npm run db:reset` 실행(데이터 초기화됨). 기존 데이터를 유지하려면 `prisma.achievement`에 새 업적을 수동 추가 후 서버 재시작.
- 결제/x402 플로우 업데이트: 결제용 서명만 받는 시뮬레이션으로 변경(별도 approve 불필요). 402 응답을 받으면 서명 후 재시도하면 됨. DB 스키마는 기존 상태 유지. 서버 재시작 후 프롬프트 → 402 → 서명 버튼 → 재시도 플로우를 확인하세요.
- Prisma 스키마 변경(업적/좋아요 등 포함)이 있으니 `cd backend && npx prisma db push`로 반영 후 서버 재시작.
