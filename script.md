# 🟦 **2. 해결책 — ‘Proof-of-Prompt(PoP)’의 핵심 개념 (1:20~2:10)**

PoP의 핵심은 아주 간단합니다. 모델 플로우 

### ✔ 유저는 모델을 선택할 수 없습니다.

어떤 모델이 응답할지 모른 채, 프롬프트를 입력합니다.

### ✔ 프롬프트는 **X402 실시간 결제**를 통해 전송됩니다.

즉, “진짜 돈을 지불하는 행동 데이터”가 남습니다.

이 데이터는 조작될 수 없습니다.

### ✔ 모델은 익명으로 응답합니다.

사용자는 브랜드를 보지 않고, 오직 응답의 질만 보게 됩니다.

### ✔ 게시판에 올리면 그때서야 모델명이 공개됩니다.

어떤 모델이 어떤 프롬프트에서 좋은 결과를 냈는지

모든 것이 투명하게 드러납니다.

이 구조 덕분에

선입견 없이 “결과 기반 평가”가 가능해집니다.

---

# 🟦 (뒤로)**. 데모 — 사용자가 경험하는 PoP 흐름 (2:10~3:30)**

이제 실제 사용 흐름을 예시로 보여드리겠습니다.

### **Step 1 — 사용자 프롬프트 입력**

사용자는 원하는 질문을 입력합니다.

예: “React에서 useEffect를 최적화하는 방법 설명해줘.”

### **Step 2 — ‘Generate’ 클릭 → X402 결제 발생**

유저는 해당 모델 호출 비용을 즉시 지불합니다.

X402는 온체인 실시간 결제 프로토콜로,

이 결제는 취소·조작이 불가능합니다.

즉, **유저의 선택은 실제 가치가 담긴 행동 데이터**가 됩니다.

### **Step 3 — 익명 모델 중 하나가 응답**

모델명은 숨겨져 있습니다.

유저는 결과만 보고 평가합니다.

### **Step 4 — ‘게시판에 올리기’ 클릭**

게시판에는

- 프롬프트
- 모델 응답
- 그때서야 공개되는 모델명
    
    이 세 가지가 등록됩니다.
    

이제 다른 사용자들이

좋아요, 댓글, 반응을 남깁니다.

여기서 중요한 포인트는:

> 게시판에서 좋은 반응을 얻는 프롬프트 =
해당 모델이 실제로 실사용자에게 좋은 응답을 제공한 모델이라는 뜻입니다.
> 

브랜드도 없고, 마케팅도 없고,

오직 응답 품질과 사용자 반응으로만 평가됩니다.

# 🟦 **4. 보상 구조 — 커뮤니티가 살아있는 이유 (3:30~4:20)**

PoP는 단순히 결과를 보여주는 플랫폼이 아니라,

**커뮤니티가 계속해서 좋은 데이터를 생성하도록 설계되어 있습니다.**

### 💰 **매주 가장 좋은 반응을 얻은 프롬프트 작성자에게 보상을 지급합니다.**

즉,

- 좋은 프롬프트를 만들고
- 익명 모델이 좋은 답변을 만들고
- 커뮤니티가 인정하면
- 유저는 실질적 보상을 받습니다.

이 구조는 자연스럽게 선순환을 만듭니다.

### **“좋은 프롬프트 → 좋은 모델 응답 → 게시판 반응 → 보상 → 더 좋은 프롬프트”**

그 결과 PoP는

**커뮤니티 주도로 AI 모델을 평가하고,
실사용 기반 고품질 데이터셋을 생성하는 시스템**이 됩니다.

---

# 🟦 **5. PoP가 만들어내는 가치 (4:20~4:50)**

### ✔ 1) 실사용 기반 모델 벤치마크

“누가 실제 돈을 내고 쓰는 모델인가?”

이 데이터가 곧 모델 품질의 증거입니다.

### ✔ 2) 편향 없는 평가

유저는 모델을 모른 채 사용하기 때문에

브랜드 이미지, 가격, 선입견이 개입할 수 없습니다.

### ✔ 3) 유저 인센티브 기반 데이터셋 생성

사용자가 만든 프롬프트·응답·반응 데이터는

그 자체로 실사용자 기반 고품질 평가셋이 됩니다.
